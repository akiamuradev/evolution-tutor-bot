EVO:LUTION Tutor Bot

Мультиплатформенный AI-репетитор: Telegram-бот, HTTP API для VK Mini App, React/Vite фронтенд, PostgreSQL-хранилище учебного прогресса, RAG-поиск по базе задач, OpenRouter-интеграция, защита от перегрузки и инструменты обработки образовательных данных.

Python backend 74 файла Python в активном backend/tools контуре
Frontend React VK Mini App на Vite и VK Bridge
Runtime 3 сервиса Docker Compose: bot, API, PostgreSQL
Data layer RAG поиск, ранжирование, прогресс, память
Python 3.11 aiogram 3 aiohttp asyncpg PostgreSQL pgvector image OpenRouter React 18 Vite VK Bridge Docker Nginx

Runtime

Запуск и жизненный цикл

В Docker Compose проект поднимает отдельный Telegram worker, отдельный HTTP API и PostgreSQL на образе pgvector/pgvector:pg16.

Сервис Команда / образ Роль
tutor-bot python -m src.bot Telegram polling, middleware, routers, AI responses, progress tracking.
tutor-api python -m src.web_api aiohttp API for VK Mini App and web clients, health, profile, chat, practice.
postgres pgvector/pgvector:pg16 Users, tasks, progress, memory, achievements, activity sessions, search indexes.
Проверка окружения Bot требует TG_BOT_TOKEN, DATABASE_URL, OPENROUTER_API_KEY; API требует DATABASE_URL и AI-ключ.
Инициализация БД Database.init() создает таблицы, добавляет колонки через ALTER TABLE IF NOT EXISTS и поднимает индексы.
Подключение сервисов services.db_pool, TaskSearch, TrendAnalyzer доступны одинаково для bot и API.
Защита входа AntiSpamMiddleware подключается к message/callback update'ам, а API использует тот же guard через _check_api_action().
Фоновая уборка Каждый час закрываются старые активные учебные сессии и пересчитывается время активности.

Tutoring Core

Единый движок ответа для разных платформ

modules/tutor_engine.py убирает дублирование: Telegram и VK/Web API проходят через одну функцию generate_tutor_response().

    Adapter->>Adapter: auth, consent, limits, anti-spam

    RAG->>DB: ranked task search
    Engine->>Guard: guarded OpenRouter request
    Guard->>AI: selected model + messages

    Guard-->>Engine: text + busy status

    Engine-->>Adapter: answer, guided flag, RAG metadata
    Adapter-->>Client: response
Компонент Что демонстрирует Файлы
Guided tutoring Бот отличает концептуальный вопрос от попытки получить готовое решение и может вести ученика подсказками. modules/tutor_intent.py, modules/prompts.py
Memory-aware prompt Ответ строится из system prompt, краткой истории, долгой памяти, релевантных прошлых фрагментов и RAG-контекста. modules/memory.py, database.py
Reusable platform logic Одинаковая логика обучения обслуживает Telegram, VK Mini App и API без привязки к UI-платформе. modules/tutor_engine.py, api/routes.py, routers/messages.py
Response hygiene Ответы очищаются, режутся на Telegram-safe chunks и сохраняются в историю с оценкой важности. modules/utils.py, database.py

AI Infrastructure

OpenRouter, маршрутизация моделей, кеш и защита дорогих запросов

AI-слой выделен в отдельные модули: клиент отвечает за вызов модели, gateway за очередь и статистику, model router за выбор режима, request guard за конкурентный доступ.

Model routing

model_router.py классифицирует вопрос как fast, standard или reasoning по длине, формулам, экзаменационным маркерам и предметным словам.

Fallback strategy

ai_client.py пробует fallback-модели и сниженный max_tokens для retryable статусов 400/408/429/5xx.

AI queue

request_guard.py держит глобальный semaphore, per-user lock, лимит ожидающих запросов и timeout ожидания свободного AI-слота.

Observability

ai_gateway.py считает started/completed/busy/cancelled/failed, inflight/model-active по типам генерации и среднее время ожидания.

LRU cache

cache.py кеширует AI-ответы по нормализованным messages и model, ограничивает размер и удаляет записи по TTL.

Cancellation UX

generation_control.py хранит активную генерацию на пользователя, умеет отменять задачу и заменять устаревшую генерацию новой. 

flowchart TD
    Q["Student question"] --> C["classify_question_complexity"]
    C --> M["select_model"]
    M --> Cache{"LRU cache hit?"}
    Cache -->|yes| Return["return cached answer"]
    Cache -->|no| Lease["AIRequestGuard acquire"]
    Lease --> Busy{"slot acquired?"}
    Busy -->|no| Soft["soft busy response"]
    Busy -->|yes| OR["OpenRouter request"]
    OR --> OK{"200 OK?"}
    OK -->|yes| Save["cache + stats"]
    OK -->|retryable| Retry["fallback model / lower tokens"]
    Retry --> OR
    OK -->|fatal| Error["error text"]
    Save --> Return

Data & RAG

База задач, полнотекстовый поиск и учебная память

Проект использует PostgreSQL как центральную систему хранения: пользователи, учебные задачи, прогресс, достижения, активные сессии, события и AI-память находятся в одной связанной модели данных.

Таблица / индекс Назначение
users Профиль, согласие, класс, режим, бесплатные лимиты, подписки, голосовой режим, счетчик запросов.
subjects, fipi_tasks Каталог предметов и задач: год, номер, сложность, тема, условие, решение, ответ, объяснение, source URL.
student_progress Ответы учеников, correctness, subject/topic, признак использования объяснения.
achievements, user_events Открытые достижения и события для метрик активности, запросов и временных условий.
ai_dialog_messages История диалога с importance, tags и ограничением последних 80 сообщений на пользователя.
ai_memory_profiles Сжатый summary и learning profile, обновляемые через AI каждые 12 сообщений.
user_sessions, user_time_stats Расчет активного времени, закрытие старых сессий, суммарная статистика.
GIN to_tsvector indexes Поиск по задачам и памяти на condition/topic/subtopic/answer/content.

TaskSearch

Умеет выдавать случайные задачи для практики, искать по теме, ключевым словам и ранжировать кандидатов по предмету, номеру задания, topic/subtopic/condition/answer.

Tutor-RAG pipeline

Анализирует запрос ученика, определяет предмет и номер задания, рассчитывает confidence и добавляет компактный контекст только при полезной уверенности.

Memory retrieval

Ищет релевантные прошлые фрагменты не из последних сообщений, сортирует по importance и собирает безопасный блок памяти для system prompt.

TrendAnalyzer

Считает экзаменационную статистику по предметам: количество задач, годы, уникальные темы, ранний и поздний год в базе.

VK Mini App & API

Второй клиент поверх той же backend-логики

VK Mini App не содержит секретов и общается с backend только через публичные API endpoints. На фронтенде реализованы чат, практика, достижения, профиль, активность, темы оформления и VK Bridge init.

Endpoint Метод Использование
/health GET Состояние API, БД, RAG, anti-spam, AI queue, generation registry, VK auth config.
/api/chat POST AI-ответ через общий generate_tutor_response(), RAG metadata, guided flag.
/api/profile GET Класс, лимиты, подписка, запросы, достижения, активность.
/api/achievements GET Сводка достижений, unlocked/locked, progress percent, rarity labels.
/api/activity GET Активное время, сегодня, всего, лучшая сессия, текущая сессия.
/api/practice/task GET Выдача случайной задачи по предмету из TaskSearch.
/api/practice/answer POST Проверка ответа, запись в student_progress, возврат решения и достижений.

VK authentication

vk_auth.py проверяет подпись launch params через HMAC SHA-256, срок действия vk_ts, user id и поддерживает dev/unsigned режимы через env flags.

Frontend architecture

App.jsx держит client state, Promise.all dashboard loading, chat composer, practice workflow, tabs, fallback achievements и форматирование API errors.

Responsive UI

styles.css реализует светлую/темную тему, мобильную адаптацию, безопасные размеры, progress bars, chat bubbles и practice cards.

Local developer loop

vite.config.js проксирует /api и /health на localhost:8080, сохраняя frontend простым в локальной разработке.

Reliability

Антиспам, нагрузка, health checks и безопасная деградация

Проект учитывает реальные ограничения: пользователи могут спамить, AI-запросы дорогие, Telegram update'ы приходят всплесками, а API должен возвращать понятные статусы занятости.

Per-action anti-spam

Отдельные профили для message, command, callback, api_chat, api_practice: min interval, burst window, max events, cooldown.

Global update limit

MAX_CONCURRENT_UPDATES ограничивает количество одновременно обрабатываемых событий в процессе.

AI queue admission

MAX_CONCURRENT_AI_REQUESTS, AI_QUEUE_WAIT_SECONDS, MAX_WAITING_AI_REQUESTS отделяют дорогую AI-нагрузку от обычных update'ов.

Admin health

Telegram /health и API /health показывают DB, RAG, cache, anti-spam, AI queue, AI gateway и generation registry.

Настройка Значение в шаблоне Смысл
MAX_CONCURRENT_UPDATES 200 Глобальный потолок обработки Telegram/API действий внутри процесса.
MAX_CONCURRENT_AI_REQUESTS 20 Одновременные запросы к LLM.
AI_QUEUE_WAIT_SECONDS 45 Сколько запрос может ждать свободный AI slot.
MAX_WAITING_AI_REQUESTS 200 Защита очереди ожидания от бесконечного роста.
WEB_API_MAX_TEXT_LENGTH 4000 Ограничение размера входного текста для API chat.
RAG_MIN_CONFIDENCE 0.22 Порог, ниже которого контекст задач не добавляется в prompt.

Security & Privacy

Практики безопасности, которые уже видны в коде

Secrets outside frontend

OpenRouter, Telegram, PostgreSQL и YooKassa параметры описаны в .env.example и не требуются во frontend runtime.

VK signature verification

Launch params проверяются HMAC-подписью, timestamp freshness и user id validation.

Consent flow

Telegram onboarding требует согласия перед обычным использованием бота; есть команды удаления данных и отзыва согласия.

Input limits

API ограничивает размер JSON client body и длину текста; anti-spam профили защищают chat/practice endpoints.

Memory safety

Профиль памяти обновляется отдельным prompt с ограничением на чувствительные данные и truncation длины записей.

Operational boundary

Nginx config проксирует только /health и /api/ на локальный API-порт.
